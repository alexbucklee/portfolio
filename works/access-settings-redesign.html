Редизайн настроек доступа

Редизайн интерфейса настроек доступа ERP-системы для вузов

10/2024

  • UX/UI
  • Прототипирование

ЮНИВУЗ — система для управления образовательным процессом в вузах. Система состоит из нескольких модулей. Каждый модуль сервиса имеет свой раздел «Настройки доступа». В данном разделе добавляются пользователи, и для них выставляются необходимые права для работы в модуле.

Исходные данные

Проблемы

  1. Нет элемента интерфейса, по которому бы раскрывались права доступа всех пользователей.
  2. Вложенные таблицы c правами доступа сильно бросаются в глаза, необходимо подумать над разгрузкой интерфейса.
  3. Отсутствуют фильтры табличного списка.
  4. Отсутствует дата добавления пользователя.
  5. Отсутствует отображение должностей и структурных подразделений пользователей.

Ход работы

Кейсы

Выписал основные пользовательские кейсы, для которых предназначен интерфейс. Определил пользовательские флоу по каждому из кейсов.

Сущности

Использовал объектно-ориентированный подход: выделил сущности и их атрибуты, а также возможные методы (действия) с этими сущностями.

Дизайн-аудит

Проанализировал текущий интерфейс приложения. Выписал основные моменты, которые необходимо исправить.

Фронтенд

Поисследовал ресурс с демо-библиотекой, которая используется в продукте. Изучил компоненты, которые будут использоваться в интерфейсе, их возможности и свойства.

Первые наброски

Приступил к поиску решения для основного интерфейса, а также вспомогательных окон по добавлению/редактированию пользователей.

Результаты

Форма просмотра прав доступа пользователя

Данные о пользователе и его правах объединены в одну табличную запись. В таблице отображаются необходимая информация о пользователе. Добавлена возможность фильтрации записей по атрибутам объектов.

Форма добавления прав пользователя

Форма для добавления нового пользователя спроектирована в виде модального окна: для отдельной задачи — отдельный интерфейс.

Форма редактирования прав пользователя

Интерфейс для редактирования прав пользователя аналогичен форме добавления пользователя.
