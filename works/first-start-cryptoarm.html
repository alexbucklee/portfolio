Проект в фигме

Доработка первого запуска приложения КриптоАРМ Mobile.

08/2024

  • UX/UI
  • Иллюстрации
  • Прототипирование

КриптоАРМ Mobile — мобильное приложение для электронной подписи и шифрования документов. Задачей было доработать и освежить онбординг приложения, чтобы улучшить пользовательский опыт первого запуска.

Задача

  • Обновить визуал онбординга: старый визуал был неактуален, т.к. в нем отображались скрины старого интерфейса.
  • Помочь пользователю в настройке приложения: перед началом работы приложение необходимо настроить: сгенерировать криптографические ключи, добавить сертификат подписи.

Предварительная работа

Анализ текущей ситуации

Старый визуал делал акцент на демонстрации интерфейса.

Обязательный шаг для генерации ключей появлялся в виде модального окна после онбординга и прерывал пользовательский сценарий.

При первом запуске пользователь не знает, что для полноценной работы приложения, его необходимо настроить, а также, что ему доступна 30-дневная лицензия продукта.

Пустые состояния интерфейса не помогают пользователю начать работу.

Поиск решения

Для решения задачи рассматривалось два основных варианта:

  1. Пошаговый подробный мастер c уточнением целей использования продукта и настройкой необходимых компонентов.
  2. Онбординг с обязательными настройками и ссылкой на дополнительные настройки в конце сценария.

Анализ конкурентов

Многие продукты в онбординге делают акцент на визуал и презентацию основных фич продукта, а не на демонстрации интерфейса.

Также многие приложения включают необходимые шаги для первичной настройки приложения в сам онбординг.

Результаты

Обновленный флоу первого запуска

Добавил два новых экрана:

  1. Обязательный экран для генерации криптографических ключей.
  2. Приветственный экран, в котором говорится о бесплатной временной лицензии и предложением пройти диагностику.

Новый визуал для онбординга

Разработал новые иллюстрации согласно общей стилистике продукта для акцентирования внимания на фичах продукта.

Пустые состояния интерфейса

Обновил пустые состояния всех интерфейсов приложения. Теперь интерфейсы сообщают, для чего они нужны и как начать с ними работу. Для привлечения внимания добавил готовые иллюстрации из десктоп-версии продукта, доработав их для темной темы.
